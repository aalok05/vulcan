Insecure Direct Object Reference (IDOR)
HIGH FIXED
File: tests/scenarios/idor_user_profile.py:17

The /api/users/<user_id>/profile endpoint returns full user data, including sensitive fields, without validating that the requesting user is authorized to access the specified user_id. This allows any authenticated user to retrieve another user's private information.

Suggested Fix:

Verify that current_user.id matches user_id or that the user has appropriate privileges before returning the profile. Remove or restrict sensitive fields such as private_notes from the response.
